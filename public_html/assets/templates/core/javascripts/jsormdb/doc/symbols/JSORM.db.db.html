clear()
Clear the database entirely. This is a journaled event and is part of the current transaction. If you wish to start afresh, use load() instead
commit(options)
Commit the current transaction. If there is a channel, and a non-nothing update mode, it will write to the store. If there is no channel, it will just commit. The commit mode is determined by the following:
  • options.mode - for this transaction
  • writeMode - default for this db instance
  • defaultWriteMode - default for all instances of the db
Parameters:
{Object} options Optional
Commit options
{Object} options.mode Optional
Which mode to use for committing, one of the static modes
{Object} options.params Optional
Parameters to pass to the channel as part of the update
{Function} options.callback Optional
Function to call when the commit is complete
{Function} options.success Optional
Function to call when the commit has succeeded
{Function} options.failure Optional
Function to call when the commit has failed
{options.Object} options.scope Optional
Scope within which to execute the callbacks
{options.Object} options.options Optional
Options to pass to the callbacks

insert(data)
Insert new data directly into the database. The parser will parse. An insert is considered part of a transaction and is logged in the journal. If you wish to start afresh, use load() instead.
Parameters:
{Object[]} data
Array of data objects to insert into the database
{boolean} isDirty()
Determine if there are any changes in the current transaction. Equivalent of getModifiedCount() > 0
Returns:
{boolean} If there are any changes
load(args)
Load new data to reinitialize this database. This is different from insert in several ways:
  1. The load (and possible replace) are not considered part of the current transaction. The current transaction is terminated, and a new transaction is started immediately after the load
  2. The load can come from either data passed directly or from the channel
  3. load is always asynchronous, whereas insert is synchronous
Parameters:
{Object} args
Arguments to the load.
{Object} args.data Optional
Raw data to load. If null, will use the defined channel and parser.
{Function} args.callback Optional
Function to call when the load is complete
{Function} args.success Optional
Function to call when the load succeeds
{Function} args.failure Optional
Function to call when the load fails
{Object} args.scope Optional
Scope within which to call the callbacks.
{Object} args.options Optional
Object with options to pass to the callback.
reject(count)
Reject a transaction. If given a count, it will reject the last count activities. If given no count, a count of 0, or a count greater than the total number of activities in this transaction, it will reject the entire transaction.
Parameters:
{Integer} count
Number of steps within the transaction to reject. If empty, 0, or greater than the total number of steps, the entire transaction will be rejected.
